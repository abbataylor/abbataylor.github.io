How to Add Your Live Projects

This guide explains exactly how to make each of your projects accessible to recruiters and your lecturer — whether they're static websites or PHP/Laravel applications.

📋 Contents

1. Understanding the two types of projects 2. Adding screenshots to any project 3. Activating live demos for static projects 4. Hosting your PHP/Laravel projects for free 5. Updating your portfolio after deployment 6. Pushing changes to GitHub

1. Understanding the two types of projects

Your projects fall into two categories. Each needs a different approach to go live:

Static Projects → Easy

Pure HTML/CSS/JavaScript — no server needed. Can go directly on GitHub Pages.

  • Countdown Timer
  • JS Calculator
  • Geo Website (TypeScript)
  • Gold Dealers Website (TypeScript)
  • Portfolio (already live)

PHP/Backend Projects → Needs hosting

Require a PHP server and MySQL database. Can't run on GitHub Pages.

  • CALIZA (Real Estate)
  • Careers Portal
  • Jetflix
  • School Report Generator
  • Student Mental Health System
  • Cargo Shipping Platform
  • IT Support Chatbot
  • Student Clearance System

2. Adding screenshots to any project

Screenshots are the quickest way to show a project — add them to any project regardless of whether it has a live demo. They appear in a gallery on the case study page.

1

Take screenshots of your running project. Use Windows Snipping Tool (Win + Shift + S) or press PrintScreen.

2

Save them as .jpg files. Name them clearly: screenshot1.jpg, screenshot2.jpg, etc.

3

Place them in the correct project folder inside:
assets/images/projects/[project-id]/
For example: assets/images/projects/caliza/screenshot1.jpg

4

Open assets/js/config.js and find your project. Update the screenshots array — remove the // comment marks and fill in the paths:

// BEFORE (commented out):
screenshots: [
  // "assets/images/projects/caliza/screenshot1.jpg",
  // "assets/images/projects/caliza/screenshot2.jpg"
],

// AFTER (active):
screenshots: [
  "assets/images/projects/caliza/screenshot1.jpg",
  "assets/images/projects/caliza/screenshot2.jpg",
  "assets/images/projects/caliza/screenshot3.jpg"
],
5

Save config.js, commit, and push. The screenshots will appear on the case study page with a lightbox gallery.

3. Activating live demos for static projects

The demo/ folder already has a placeholder page for each static project. Replace the placeholder with your actual project files.

Countdown Timer & JS Calculator

1

Open the project folder on your computer (e.g. your countdown-timer folder).

2

Copy all files from the project into the matching demo folder:
demo/countdown-timer/ or demo/js-calculator/

3

Delete the placeholder index.html that's already in the folder, and replace it with your project's index.html.

4

Commit and push. The live demo will be accessible at:
abbataylor.github.io/demo/countdown-timer/

TypeScript Projects (Geo Website & Gold Dealers)

⚠️ Important: TypeScript must be compiled to JavaScript before it can run in a browser. GitHub Pages cannot run .ts files directly.
1

Open a terminal in your project folder and run: tsc (this compiles TypeScript → JavaScript)

2

Copy all files including the compiled .js files into demo/geo-website/ or demo/gold-dealers/

3

Delete the placeholder index.html and replace with your project's main HTML file (renamed to index.html).

4

Commit and push.

4. Hosting your PHP/Laravel projects for free

These platforms support PHP + MySQL for free and are suitable for showcasing your projects to employers and lecturers:

InfinityFree

infinityfree.com

Completely free PHP + MySQL hosting. Good for plain PHP projects. Upload your files via FTP.

✓ Free forever   ✓ PHP + MySQL   ✓ FTP upload

Railway

railway.app

Best for Laravel apps. Free trial available. Connect your GitHub repo and deploy automatically.

✓ Laravel support   ✓ GitHub deploy   ✓ Free trial

000webhost

000webhost.com

Free PHP hosting by Hostinger. Suitable for plain PHP projects. Easy file manager interface.

✓ Free   ✓ PHP + MySQL   ✓ Easy upload

Render

render.com

Free tier for web services. Good for Laravel projects deployed from GitHub. Professional-looking URLs.

✓ Laravel & PHP   ✓ GitHub connect   ✓ Free tier

Recommended approach for plain PHP projects (XAMPP → InfinityFree)

1

Go to infinityfree.com and create a free account.

2

Create a new hosting account — you'll get a free subdomain like yourproject.infinityfreeapp.com

3

In the InfinityFree control panel, go to MySQL Databases, create a database and note down the host, database name, username, and password.

4

Use phpMyAdmin (available in the control panel) to import your SQL file and create all your tables.

5

Update your project's database connection file (e.g. config.php or .env) with the new database credentials.

6

Use the File Manager in the InfinityFree control panel to upload all your project files to the htdocs folder.

7

Visit your subdomain URL to test the project is working.

Recommended approach for Laravel projects (Railway)

1

Push your Laravel project to a private GitHub repository (it won't expose the source code publicly).

2

Go to railway.app and sign in with GitHub.

3

Click New Project → Deploy from GitHub repo and select your Laravel repo.

4

Add a MySQL service to the project, then connect it to your app using the environment variables Railway provides.

5

Set your environment variables (APP_KEY, DB_HOST, etc.) in the Railway dashboard under Variables.

6

Railway will give you a public URL — copy it and add it to config.js as the liveDemo link.

💡 Tip: For private projects that aren't deployed yet, adding screenshots is enough. Recruiters understand that private work can't always be shown live — the case study and screenshots prove the project exists and works.

5. Updating your portfolio after deployment

Once a project is live, open assets/js/config.js and update its liveDemo field:

// Find the project in the projects array, e.g. caliza
{
  id: "caliza",
  ...
  liveDemo: "https://caliza.infinityfreeapp.com",   // ← paste your URL here
  ...
}

Save the file, then follow Section 6 to push the change to GitHub.

⚠️ Important: Keep your database credentials in .env files only — never commit them to GitHub. Add .env to your .gitignore file before pushing any backend project.

6. Pushing changes to GitHub

After adding screenshots, demo files, or updating config.js, run these commands in your terminal from the portfolio folder:

cd "c:\Users\256\Desktop\abbataylor.github.io"

git add .
git commit -m "Add screenshots and live demo links"
git push origin master

GitHub Pages will update automatically within 1–2 minutes. Your recruiter or lecturer will see the changes immediately after that.
